📋 Enterprise PMS 모듈별 상세 기능 명세서 (Functional Specification)



본 문서는 PMS Master Directive의 아키텍처 및 비즈니스 목표를 바탕으로, 각 모듈에서 실제로 구현되어야 하는 사용자 인터페이스(UI)와 세부 기능(Features)을 기획한 명세서입니다.



Module A: 수주 기획 및 파이프라인 관리 (Pre-Sales \& Bidding)



목적: 엑셀로 산재된 제안 견적을 시스템화하고, 입찰 이익률을 시뮬레이션하여 전략적 수주를 지원합니다.



1\. 엑셀 파싱 기반 견적 자동 등록 (Smart Import)



기능: 제안서용 견적 엑셀 파일을 업로드하면, 시스템이 기계, 전/계장, 구매품 등 대분류와 하위 항목, 단가, 수량을 자동으로 읽어들여 웹 그리드에 렌더링합니다.



UI/UX: 파일 Drag \& Drop 영역, 업로드 성공/실패(오류 셀 위치 안내) 모달창.



2\. 수주 파이프라인 칸반 보드 (Kanban View)



기능: 진행 중인 모든 수주 프로젝트를 단계별(타겟 발굴 ➔ 제안 준비 ➔ 입찰 ➔ 수주/실패)로 나누어 칸반 보드 형태로 시각화합니다.



UI/UX: 마우스 Drag \& Drop으로 카드의 단계를 이동시킬 수 있으며, 카드 클릭 시 우측 슬라이드 패널로 상세 원가 내역이 열립니다.



3\. 목표 이익률 시뮬레이터



기능: '목표 이익률(%)' 입력 필드에 숫자를 입력하면, 사내 간접비 및 판관비 요율을 반영하여 자동으로 '타겟 입찰가'를 역산하여 보여줍니다.



UI/UX: 변경되는 수치가 실시간으로(Dynamic) 반영되는 Summary 위젯.



4\. 입찰 히스토리 및 버전 비교 (Version Diff)



기능: 견적 금액이나 항목이 수정될 때마다 Save as New Version 버튼을 통해 v1.0, v1.1로 스냅샷을 저장합니다.



UI/UX: 두 버전을 선택하여 비교하는 'Compare 모드' 제공. 금액이 증가한 항목은 빨간색(+), 감소한 항목은 파란색(-) 텍스트로 차이점(Diff)을 직관적으로 표시합니다.



Module B: 실행원가 및 예산 통제 시스템 (Cost Tracking)



목적: 복잡한 '실행예산서'를 웹 그리드로 이관하고, 비용의 집행 현황을 1원 단위까지 실시간으로 통제합니다.



1\. 다단 헤더 실행예산 엑셀 업로드



기능: 실행예산서 (2) 형태의 엑셀 업로드 시, '최초 실행원가', '네고 반영가', '1\~12월 사용계획' 등의 병합된 다단 헤더를 시스템이 인식하여 DB에 정규화하여 저장합니다.



2\. 틀고정 기반 대용량 데이터 그리드



기능: 수백 개의 WBS 항목과 예산 정보가 담긴 표를 조회합니다.



UI/UX: 상단 헤더(항목, 업체명, 계정명 등)와 좌측 1\~3열을 고정(Freeze)하여 가로/세로 스크롤 시에도 어떤 항목의 예산인지 항상 확인할 수 있습니다. 가상 스크롤(Virtual Scroll)을 적용해 지연 없이 렌더링합니다.



3\. 실시간 비용 트래킹 및 초과 경보 (Cost Control)



기능: (계획 예산) - (기사용금액 + 금번사용금액 + 잔여예정금액) 공식을 통해 '원가 절감/초과액'을 실시간 자동 계산합니다.



UI/UX: 계산 결과값이 마이너스(초과)가 되는 순간 해당 셀의 배경색을 옅은 붉은색으로 변경하고 경고 아이콘(⚠️)을 띄워 시각적으로 즉시 인지하도록 합니다.



Module C: 전자 결재 워크플로우 (Digital Approval)



목적: 예산 집행의 투명성을 확보하고, 기성 품의 승인 시 재무 데이터를 자동 업데이트합니다.



1\. 동적 결재선 및 폼 선택



기능: 내부 품의, 협력사 기성, 고객사 청구 등 템플릿을 선택하고 결재자/참조자를 지정합니다.



UI/UX: 부서 조직도 기반의 결재자 검색 모달, 부재중인 결재자를 위한 '대결자 자동 지정' 안내 기능.



2\. 원가 데이터 자동 연동 기성 품의



기능: 품의서 작성 시 금액을 수기로 입력하지 않고, \[Module B]의 예산 그리드에서 집행할 항목을 체크(☑)하여 '품의 상신'을 누르면 금액과 내역이 품의서에 자동 매핑됩니다.



3\. 승인 트리거 자동 업데이트 (Auto-sync)



기능: 최종 결재자가 '승인' 버튼을 누르면, 백엔드에서 트랜잭션이 발생하여 해당 항목의 금번사용금액을 0으로 비우고 그 액수만큼 기사용금액에 더해줍니다. (실패 시 자동 롤백).



4\. 모바일 결재 (Mobile UI)



기능: 스마트폰에서 접속 시 결재 대기 문서를 목록으로 보고, 첨부된 영수증/사진을 확인 후 터치 한 번으로 승인/반려할 수 있습니다.



Module D: 인력 배치 및 리소스 통제 (Resource Allocation)



목적: 직원들의 투입률을 관리하여 과부하를 방지하고, 적재적소에 인력을 배치합니다.



1\. 전사 리소스 히트맵 (Heatmap Board)



기능: 세로축에는 부서/직원명, 가로축에는 주차(Week) 또는 월(Month)을 두고 개인별 프로젝트 투입률(%)을 색상으로 표시합니다.



UI/UX: 투입률 0\~50%는 녹색, 51\~99%는 노란색, 100% 초과는 빨간색으로 셀 색상을 자동 변경합니다.



2\. 프로젝트 인원 배분 및 충돌 감지



기능: PM이 특정 인원을 자신의 PJT에 투입(예: 3\~5월, 50% 투입) 설정할 때, 해당 직원이 이미 타 PJT에 배정되어 합산 100%가 넘어가면 저장을 차단하고 팝업으로 경고합니다.



3\. 필요 직무 누락 알림



기능: 시공 단계 진입 시 현장 소장이나 안전 관리자가 시스템에 배정되지 않은 경우, PM에게 "필수 인력이 누락되었습니다"라는 경고 플래그를 띄웁니다.



Module E: 간트차트 및 진도 관리 (Gantt Schedule)



목적: WBS(작업 분할 구조도)를 시각화하여 지연된 공정을 직관적으로 파악합니다.



1\. 인터랙티브 간트 뷰어 (Interactive Gantt)



기능: 프로젝트의 작업 일정(설계, 구매, 시공 등)을 바(Bar) 차트로 시각화합니다.



UI/UX: 바의 양끝을 마우스로 드래그하여 일정을 늘리거나 줄일 수 있고, 전체 바를 드래그하여 시작일을 이동시킬 수 있습니다.



2\. 계획(Baseline) vs 실적(Actual) 비교



기능: 최초 승인된 계획 일정을 반투명한 배경 바(Baseline)로 고정시키고, 실제 진행률을 내부의 진한 컬러 바로 겹쳐 보여주어 지연 여부를 즉시 파악합니다.



3\. 자동 종속성 밀림 현상 처리 (Dependencies)



기능: 선행 작업(예: 설계 완료)에 후행 작업(예: 자재 발주)을 화살표로 연결해 둡니다. 선행 작업이 1주일 지연되어 일정이 수정되면, 연결된 모든 후행 작업들의 시작일도 자동으로 1주일씩 뒤로 밀립니다.



Module F: 경영진 대시보드 및 알림 센터 (Dashboard \& Alerts)



목적: 모든 모듈의 데이터를 취합하여 인사이트를 제공하고 즉각적인 조기 경보를 울립니다.



1\. KPI 및 재무 요약 위젯 (Dashboard Widgets)



기능: \* '26년 전사 수주 목표액 대비 달성률 (도넛 차트).



매출 대비 제조원가, 판관비 비율 및 예상 영업이익 (누적 막대 차트).



부서별 인력 가동률 요약 표.



2\. 지능형 리스크 조기 경보 (Proactive Alerts)



기능: 백그라운드 스케줄러가 매일 자정에 데이터를 스캔합니다.



조건 1: 특정 항목의 실제 집행액 + 잔여 예정액이 계획 예산을 1원이라도 초과함.



조건 2: 크리티컬 패스(Critical Path) 상의 일정이 2주 이상 지연됨.



결과: 위 조건 충족 시 다음날 아침 경영진 및 PM 접속 시 시스템 화면 상단에 붉은색 배너 알림 및 사내 이메일 자동 발송.



3\. 맞춤형 엑셀 리포트 추출 (Custom Export)



기능: 대시보드에서 필터링한 데이터(예: "진행 중인 모든 수처리 프로젝트의 현재까지 비용 요약")를 엑셀 보고서 포맷(장표 형태)으로 변환하여 1-Click 다운로드 기능을 제공합니다.
