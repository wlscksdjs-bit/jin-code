\# Phase 2: 실행 예산 관리 및 결재 워크플로우 연동



\## 1. 개요

수주가 확정된 프로젝트의 실행 예산을 편성하고, 실제 비용 집행 내역(기사용, 당회, 잔여)을 추적하며, 이를 기반으로 전자 결재를 진행하는 핵심 재무 모듈을 개발한다.



\## 2. 작업 지시 내용



\### 2.1 실행 예산 및 비용 트래킹 (Module B)

\* \*\*CostCategory 모델\*\*: 인건비, 자재비, 외주비(협력사별) 등 세부 항목 관리.

\* \*\*Budget 모델\*\*: 항목별 실행 예산(계획) 금액 등록.

\* \*\*Expense 모델\*\*: 실제 집행 비용 기록.

\* \*\*핵심 비즈니스 로직\*\*:

&#x20;   \* `사용예정금액(잔여)` = `계획(실행예산)` - (`기사용금액` + `금번사용금액`)

&#x20;   \* `원가 절감/초과 액수` 계산 후, 예산 초과 시 즉각적인 경고(Exception) 처리 및 UI 하이라이트(Red/Blue) 응답용 데이터 포맷팅.



\### 2.2 전자 결재 및 기성 관리 (Module C)

\* \*\*Approval 모델\*\*: 결재 문서 종류(내부 품의, 협력사 기성, 고객사 청구), 결재 상태(상신, 검토중, 승인, 반려), 결재선(Line) 정보 포함.

\* \*\*연동 로직\*\*: 

&#x20;   \* 비용 집행(Expense) 입력 시 반드시 Approval 프로세스를 태우도록 강제.

&#x20;   \* 승인(Approved) 상태로 변경되는 순간 트랜잭션을 발생시켜 `금번사용금액`을 `기사용금액`으로 이관하고 `잔여금액`을 재계산.

\* \*\*테스트 조건\*\*: 결재 승인 과정에서 DB 트랜잭션 오류 발생 시 금액이 롤백(Rollback)되는지 엄격히 검증.



\## 3. 산출물

\* 예산 편성 및 실시간 비용 추적 API.

\* 항목별 원가 분석표 UI (엑셀 스타일의 데이터 그리드 권장).
