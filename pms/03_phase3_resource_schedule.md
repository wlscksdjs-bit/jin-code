\# Phase 3: 리소스 할당 및 WBS/일정 관리



\## 1. 개요

프로젝트를 수행할 인력을 배치하여 중복/부족을 관리하고, 주요 마일스톤(설계, 구매, 시공 등)에 대한 주차별 일정을 간트차트로 시각화한다.



\## 2. 작업 지시 내용



\### 2.1 인력 배치 및 리소스 관리 (Module D)

\* \*\*ResourceAllocation 모델\*\*: 프로젝트 - 사용자 - 투입 기간(시작/종료) - 투입률(%, M/M) 매핑.

\* \*\*충돌 감지 로직\*\*: 특정 직원이 같은 기간에 여러 프로젝트에 투입되어 합산 투입률이 100%를 초과할 경우, API 단에서 Warning을 반환.

\* \*\*UI 요구사항\*\*: 특정 주차/월별로 인력 투입 현황을 보여주는 히트맵(Heatmap) 형태의 리소스 보드 구현.



\### 2.2 공정 및 일정 관리 (Module E)

\* \*\*Task(WBS) 모델\*\*: 상위/하위 작업 트리 구조, 시작/종료일, 담당자, 진행률(%), 선행 작업(Dependency) 필드 구성.

\* \*\*간트차트 연동\*\*: 프론트엔드 간트차트 라이브러리(예: DHTMLX, Frappe Gantt 등)와 통신할 수 있는 JSON 데이터 포맷 제공.

\* \*\*자동화 로직\*\*: 선행 작업의 종료일이 지연 업데이트될 경우, 의존성이 걸린 후행 작업들의 시작/종료일이 자동으로 Push되는 로직 구현.

\* \*\*테스트 조건\*\*: 계획된 일정 대비 실제 진행률(%)이 현저히 낮을 경우 지연 플래그(Flag)가 정상 작동하는지 확인.
