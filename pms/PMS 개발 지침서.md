\# 프로젝트 관리 시스템 (PMS) 통합 개발 지침서



\## 1. 프로젝트 개요

\* \*\*목적\*\*: 기존 엑셀(`pjt 관리.xlsx`) 수기 관리의 한계를 극복하고, 수주 전 기획 단계부터 수주 후 실행원가 관리, 인력/일정 관리, 품의 결재까지 유기적으로 연동되는 통합 시스템 구축.

\* \*\*주요 타겟 환경\*\*: 데스크탑(PC) 및 외부 현장용 모바일(스마트폰/태블릿) 반응형 웹.

\* \*\*핵심 가치\*\*: 데이터 누적 및 버전 관리, 실시간 원가 통제, 리소스 최적화, 직관적인 경영진 의사결정 지원.



\## 2. 시스템 아키텍처 및 기술 스택 (권장)

\* \*\*Backend\*\*: Python (Django REST Framework 또는 FastAPI) - 안정적인 권한 관리 및 복잡한 비즈니스 로직(원가 계산 등) 처리.

\* \*\*Frontend\*\*: React.js 또는 Vue.js + TailwindCSS (반응형 UI 및 간트차트 라이브러리 연동).

\* \*\*Database\*\*: PostgreSQL (버전 관리 및 트랜잭션 무결성 확보).

\* \*\*기타\*\*: Celery/Redis (이상 감지 백그라운드 작업 및 실시간 알림).



\## 3. 핵심 모듈별 상세 요구사항 및 작업 지시



\### Module A: 수주 기획 및 입찰 관리 (대기/관리 시트 대체)

1\. \*\*기능 정의\*\*:

&#x20;  - 올해 예상 프로젝트 파이프라인 등록 (고객사, 프로젝트명, 예상 규모 등).

&#x20;  - 원가 분석 및 목표 이익률에 따른 전략적 입찰가 산출 시뮬레이션.

&#x20;  - \*\*버전 관리 (Version Control)\*\*: 입찰가 및 원가 분석 내역이 수정될 때마다 `v1.0`, `v1.1` 형태로 스냅샷을 저장하여 히스토리 추적 가능하도록 구현.

2\. \*\*테스트 지침\*\*: 금액 변경 후 저장 시 이전 버전 데이터가 손상 없이 보존되는지 검증.



\### Module B: 실행 원가 및 예산 통제 (Cost Management)

1\. \*\*기능 정의\*\*:

&#x20;  - 수주 확정 시 '실행원가' 예산 편성 기능.

&#x20;  - 항목별(인건비, 자재비, 외주비 등) 및 다수 협력 업체별 발주 금액 등록.

&#x20;  - \*\*비용 트래킹 로직\*\*:

&#x20;    - `계획(실행예산)` vs `기사용금액` + `금번사용금액(당월/당회)` + `사용예정금액(잔여)`

&#x20;    - `예상 최종 금액` = 기사용 + 금번사용 + 사용예정

&#x20;    - `원가 절감/초과 여부` = 계획 - 예상 최종 금액 (적자 시 붉은색, 흑자 시 푸른색 하이라이트).

2\. \*\*테스트 지침\*\*: 잔여 예산 초과하여 '금번사용금액' 기입 시 경고 팝업 및 결재선 승인 요구 프로세스 작동 여부 테스트.



\### Module C: 전자 결재 및 기성 관리 (Workflow)

1\. \*\*기능 정의\*\*:

&#x20;  - Module B의 비용 데이터를 기반으로 한 결재 상신.

&#x20;  - \*\*유형별 폼\*\*: 1) 해당 월 사용분 내부 품의, 2) 협력사 기성 지급 품의, 3) 고객사 청구(기성) 품의.

&#x20;  - 상신 -> 검토 -> 승인/반려 상태 변경 및 알림.

2\. \*\*테스트 지침\*\*: 기성 승인 완료 시 Module B의 '금번사용금액'이 '기사용금액'으로 자동 이관되는지 트랜잭션 롤백 테스트 포함.



\### Module D: 인력 배치 및 리소스 관리 (Resource Allocation)

1\. \*\*기능 정의\*\*:

&#x20;  - PM, 설계, 공사 담당자 등 롤(Role)별 인원 투입 계획 수립 (M/M 기준).

&#x20;  - \*\*인력 중복/부족 관리\*\*: 특정 기간에 한 직원의 투입률이 100%를 초과하거나, 프로젝트에 필수 포지션이 비어있을 경우 시각적 경고(히트맵 형태).

2\. \*\*테스트 지침\*\*: A프로젝트와 B프로젝트에 동일 기간, 동일 인원 100% 중복 투입 시도 시 차단 및 알림 발생 확인.



\### Module E: 공정 및 일정 관리 (Gantt Chart \& WBS)

1\. \*\*기능 정의\*\*:

&#x20;  - 설계, 구매, 시공, 시운전 등 주요 마일스톤 주차별 WBS 작성.

&#x20;  - 간트차트(Gantt Chart) UI 적용 (Drag \& Drop 일정 변경).

&#x20;  - 계획 일정 vs 실제 진행률(%) 한눈에 비교.

2\. \*\*테스트 지침\*\*: 선행 작업(예: 설계) 지연 시 후행 작업(예: 시공) 일정이 자동 밀림 처리 또는 경고 알림 표시 확인.



\### Module F: 경영진 대시보드 및 이상 감지 알림

1\. \*\*기능 정의\*\*:

&#x20;  - 전체 PJT 요약 (수주율, 전체 원가 대비 집행률, 지연 중인 PJT 등).

&#x20;  - \*\*이상 감지 (Anomaly Detection)\*\*: 원가 초과 예상, 마일스톤 2주 이상 지연, 핵심 인원 이탈 등 리스크 발생 시 대시보드 상단 적색 경고 및 푸시/이메일 알림창 팝업.

2\. \*\*테스트 지침\*\*: 임의로 데이터 조작하여 원가 오버런 상황 생성 후 알림창 즉각 팝업 확인.



\### Module G: 권한 관리 및 시스템 환경 (RBAC \& Mobile)

1\. \*\*기능 정의\*\*:

&#x20;  - \*\*계정별 권한(RBAC)\*\*: 최고관리자, 경영진(조회/승인), PM(수정/작성), 일반팀원(담당업무 조회/진행률 업데이트).

&#x20;  - \*\*모바일 최적화\*\*: 스마트폰으로 현장에서 사진 첨부, 기성 내역 확인 및 간단한 결재 처리가 가능하도록 Responsive Design 적용.



\## 4. 데이터베이스 설계 지침 (Core ERD Concepts)

\* \*\*Users\*\*: 사용자 정보, 권한 레벨, 소속 팀.

\* \*\*Projects\*\*: PJT 기본 정보, 수주/실행 상태, 버전 ID.

\* \*\*Budgets \& Expenses\*\*: 버전별 항목 예산, 발주처, 집행 내역(기사용, 당회, 잔여).

\* \*\*Schedules\*\*: WBS, 시작/종료일, 진행률, 선/후행 관계.

\* \*\*Allocations\*\*: PJT-사용자 매핑, 투입 기간, 투입률(%).

\* \*\*Approvals\*\*: 결재 문서 정보, 결재선, 상태값.



\## 5. 개발 및 테스트 진행 단계 (Phase Plan)

\* \*\*Phase 1\*\*: DB 모델링 및 Module A (수주 기획/버전 관리) + Module G (계정/권한) 구현.

\* \*\*Phase 2\*\*: Module B (실행원가 관리) + Module C (전자결재) 연동 로직 완성.

\* \*\*Phase 3\*\*: Module D (인력 관리) + Module E (간트차트) 시각화 작업.

\* \*\*Phase 4\*\*: Module F (대시보드/알림) 구축 및 모바일 UI 최적화.

\* \*\*Phase 5\*\*: 통합 E2E(End-to-End) 테스트 및 버그 픽스. (모든 데이터 흐름의 유기적 연동 확인).
